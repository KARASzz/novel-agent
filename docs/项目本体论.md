# 短剧项目 — 本体论模型

> 本文件定义了短剧工业化生产线中所有核心实体、属性与关系。
> 用于指导系统架构、数据流设计与代码实现。

---

## 1. 实体树形图

```
短剧项目 (ShortDramaProject)
│
├── 📦 1. 知识层 (Knowledge Layer)
│   ├── 1.1 RAG 知识库 [Google Vertex AI Search]
│   │   ├── 1.1.1 市场研究 (MarketResearch)
│   │   │   ├── 爆款题材趋势
│   │   │   ├── 观众画像（年龄/性别/偏好）
│   │   │   ├── 平台数据（红果/抖音/快手）
│   │   │   └── 盈利模式与付费转化率
│   │   │
│   │   ├── 1.1.2 写作方法论 (WritingMethodology)
│   │   │   ├── 结构模板（三幕式/四层结构）
│   │   │   ├── 爽点设置技巧
│   │   │   ├── 节奏控制规则
│   │   │   ├── 台词写作规范
│   │   │   └── 卡点/付费点设计
│   │   │
│   │   ├── 1.1.3 平台规则 (PlatformRules)
│   │   │   ├── 红果短剧审核标准
│   │   │   ├── 内容禁忌清单
│   │   │   ├── 格式要求（时长/字数/集数）
│   │   │   └── 过稿评价维度
│   │   │
│   │   ├── 1.1.4 案例库 (CaseLibrary)
│   │   │   ├── 爆款剧拆解（结构/卡点/钩子）
│   │   │   ├── 失败案例分析
│   │   │   └── 同题材竞品对比
│   │   │
│   │   └── 1.1.5 文档元数据 (DocMetadata)
│   │       ├── source_url
│   │       ├── published_date
│   │       ├── rag_score（RAG 适用性评分）
│   │       ├── category（自动分类标签）
│   │       └── last_updated
│   │
│   └── 1.2 搜索管道 (Search Pipeline)
│       ├── Tavily API 搜索器
│       │   ├── 关键词模板库
│       │   ├── 搜索深度控制（basic/advanced）
│       │   └── 结果去重与合并
│       │
│       └── 内容清洗器 (ContentCleaner)
│           ├── 正文提取（去广告/导航/噪声）
│           ├── 关键信息结构化
│           ├── 分段与摘要生成
│           └── RAG 适配格式输出（Markdown）
│
│
├── 📝 2. 创作层 (Creation Layer)
│   │
│   ├── 2.1 创意生成 (Idea Generation)
│   │   ├── 2.1.1 题材选择
│   │   │   ├── 都市复仇 / 马甲打脸
│   │   │   ├── 甜宠闪婚 / 虐恋
│   │   │   ├── 战神归来 / 逆袭
│   │   │   └── 穿越 / 重生
│   │   │
│   │   ├── 2.1.2 核心设定 (Core Premise)
│   │   │   ├── 核心误会 (CoreMisunderstanding)
│   │   │   │   ├── 类型：身份误会 / 情感误会 / 信息差
│   │   │   │   └── 描述：一句话定义
│   │   │   │
│   │   │   ├── 观众期待 (AudienceExpectation)
│   │   │   │   └── 期待被满足的节点列表
│   │   │   │
│   │   │   └── 一句话梗概 (Logline)
│   │   │
│   │   └── 2.1.3 Prompt → LLM 生成创意 JSON
│   │       └── RAG 上下文注入（市场数据 + 方法论）
│   │
│   ├── 2.2 人物设计 (Character Design)
│   │   ├── 主角 (Heroine / Hero)
│   │   │   ├── name, age, personality
│   │   │   ├── core_desire（核心欲望）
│   │   │   ├── external_conflict（外部冲突）
│   │   │   ├── internal_conflict（内心冲突）
│   │   │   └── arc（人物弧线）
│   │   │
│   │   ├── 反派 (Antagonist)
│   │   │   ├── name, role, motivation
│   │   │   └── 与主角的对立关系定义
│   │   │
│   │   └── 配角 (Supporting Cast)
│   │       ├── 功能性角色（推动剧情/制造误会/提供爽点）
│   │       └── 关系网定义
│   │
│   ├── 2.3 集纲规划 (Episode Outline)
│   │   ├── 总集数（60-100集）
│   │   ├── 分段结构
│   │   │   ├── 第1-3集：黄金开场（背景/人物/主线）
│   │   │   ├── 第4-14集：期待累积（小反转/打脸）
│   │   │   ├── 第15集：第一个付费卡点
│   │   │   ├── 第16-30集：主线推进 + 副线交织
│   │   │   ├── 第31-50集：中段高潮 + 大反转
│   │   │   └── 第51-尾集：终极对决 + 收尾
│   │   │
│   │   └── 单集结构 (PerEpisode)
│   │       ├── episode_number
│   │       ├── title
│   │       ├── core_conflict（本集核心冲突）
│   │       ├── hook（结尾钩子/悬念）
│   │       ├── is_paywall_episode（是否付费卡点集）
│   │       └── cliffhanger_type（钩子类型）
│   │           ├── 悬念型（即将揭晓）
│   │           ├── 反转型（出乎意料）
│   │           ├── 冲突升级型（矛盾激化）
│   │           └── 情绪临界型（憋屈/爽感极限）
│   │
│   └── 2.4 场次编写 (Scene Writing)
│       ├── 草稿输入层
│       │   ├── 人类友好模板 (DraftTemplate)
│       │   │   ├── `### 地点 时间` → 场景标记
│       │   │   ├── `【动作描写】` → 动作
│       │   │   ├── `人名：台词` → 对白
│       │   │   └── `人名（独白）：内容` → 内心独白
│       │   │
│       │   └── LLM 解析器
│       │       ├── 实体识别（场景/人物/动作/台词）
│       │       ├── 元数据补全（场景号/时长/情绪标签）
│       │       └── 输出：结构化 JSON
│       │
│       └── 场次数据结构 (SceneData)
│           ├── scene_id: "1-1"（集数-场次）
│           ├── time: "日/夜/晨/昏"
│           ├── location_type: "内/外"
│           ├── location: "卧室"
│           ├── characters: ["谢宴礼", "楼阮"]
│           └── plots[]
│               ├── type: "action" | "dialogue" | "monologue"
│               ├── character: string | null
│               ├── content: string
│               ├── emotion: string
│               ├── duration_sec: int
│               └── is_cliffhanger: bool
│
│
├── 🔧 3. 处理层 (Processing Layer)
│   │
│   ├── 3.1 Python 渲染引擎 (RenderEngine)
│   │   ├── 输入：结构化 JSON（来自 2.4）
│   │   ├── 处理：
│   │   │   ├── 场景号自动生成与递增
│   │   │   ├── 时长计算（语速模型）
│   │   │   ├── 工业格式渲染
│   │   │   └── 商业规则校验
│   │   └── 输出：工业标准剧本文本
│   │
│   ├── 3.2 格式渲染器 (FormatRenderer)
│   │   ├── 目标格式：《玫瑰冠冕》工业标准
│   │   │   ├── 索引行：`1-1 夜 内 卧室`
│   │   │   ├── 人物行：`人物：谢宴礼、楼阮。`
│   │   │   ├── 动作行：`▲谢宴礼和楼阮在床上拥吻`
│   │   │   ├── 对白行：`谢宴礼：告诉我，我是谁`
│   │   │   └── 旁白行：`楼阮OS：我追了你十几年...`
│   │   │
│   │   └── 渲染规则
│   │       ├── 每场之间空一行
│   │       ├── 动作与对白交错排列
│   │       ├── 卡点集末尾标注 [付费卡点]
│   │       └── 每集末尾附统计信息
│   │
│   ├── 3.3 格式校验器 (FormatValidator)
│   │   ├── 校验维度
│   │   │   ├── 结构完整性（索引/人物/动作/对白是否齐全）
│   │   │   ├── 格式合规性（▲标记/索引格式/标点规范）
│   │   │   ├── 内容合理性（台词长度/时长范围/冲突密度）
│   │   │   └── 商业规则（钩子存在/付费卡点标注/前三集定律）
│   │   │
│   │   └── 输出：校验报告 JSON
│   │       ├── is_valid: bool
│   │       ├── errors[]（致命错误）
│   │       ├── warnings[]（建议修改）
│   │       └── format_score: int (0-100)
│   │
│   └── 3.4 批量处理器 (BatchProcessor)
│       ├── 输入：草稿文件夹（多个 .txt）
│       ├── 逐文件：草稿 → JSON → 渲染 → 校验
│       ├── 输出：成品剧本文件夹 + 汇总报告
│       └── 错误处理：失败文件记录 + 跳过继续
│
│
├── 📤 4. 输出层 (Output Layer)
│   │
│   ├── 4.1 剧本文件 (ScriptFile)
│   │   ├── 格式：纯文本 / Markdown / Word
│   │   ├── 命名：`{剧名}_第{X}集_{日期}.txt`
│   │   └── 内容：工业标准格式 + 统计信息
│   │
│   ├── 4.2 数据报表 (DataReport)
│   │   ├── 单集统计（时长/场次数/台词数/冲突点）
│   │   ├── 全季统计（总时长/总集数/卡点分布）
│   │   └── 校验汇总（通过率/常见问题）
│   │
│   └── 4.3 知识库更新 (KnowledgeUpdate)
│       ├── 本次生成的经验数据回写 RAG
│       ├── 成功模式记录（有效的误会/卡点/钩子）
│       └── 失败标记（校验不通过的原因分类）
│
│
└── ⚙️ 5. 基础设施层 (Infrastructure)
    │
    ├── 5.1 LLM 服务
    │   ├── 主力模型（GLM-5 / MiniMax / 其他）
    │   ├── 用途：创意生成 / 草稿解析 / JSON 输出
    │   └── 调用方式：API Key + Prompt 模板
    │
    ├── 5.2 RAG 服务 [Google Vertex AI Search]
    │   ├── 数据存储：结构化文档（Markdown/JSON）
    │   ├── 索引构建：自动向量化 + 元数据过滤
    │   ├── 检索方式：语义搜索 + 关键词混合
    │   └── 返回格式：相关文档片段 + 相关度评分
    │
    ├── 5.3 搜索服务 [Tavily API]
    │   ├── 功能：实时网络搜索
    │   ├── 输出：标题 + 摘要 + 全文 + 元数据
    │   └── 频率：按需 / 定期更新知识库
    │
    ├── 5.4 Python 运行环境
    │   ├── 依赖：requests, json, re, datetime
    │   ├── 可选：python-docx（Word 输出）, markdown
    │   └── 部署：本地运行 / 容器化
    │
    └── 5.5 工作区 (Workspace)
        ├── drafts/          ← [输入] 人类写的草稿
        ├── knowledge_base/  ← [RAG] 知识库文档
        ├── templates/       ← [模板] Prompt 模板 + 草稿模板
        ├── core_engine/     ← [逻辑] 核心处理脚本 (parser/renderer/validator)
        ├── reports/         ← [反馈] 校验报告 + 统计报表
        ├── scripts_output/  ← [成品] 生成的工业标准剧本
        └── logs/            ← [日志] 运行与错误日志
```

## 3. 实体属性详表

### 3.1 CorePremise（核心设定）

| 属性 | 类型 | 说明 |
|------|------|------|
| genre | string | 题材类型（都市复仇/甜宠闪婚/战神归来...） |
| misunderstanding_type | enum | 身份误会 / 情感误会 / 信息差 |
| misunderstanding_desc | string | 一句话描述核心误会 |
| audience_expectation | string | 观众最期待看到什么 |
| logline | string | 一句话梗概 |
| target_episodes | int | 目标总集数（60-100） |
| paywall_episode | int | 第一个付费卡点所在集数 |

### 3.2 Character（人物）

| 属性 | 类型 | 说明 |
|------|------|------|
| name | string | 姓名 |
| role | enum | protagonist / antagonist / supporting |
| age | int | 年龄 |
| personality | string | 性格特征 |
| core_desire | string | 核心欲望 |
| external_conflict | string | 外部冲突 |
| internal_conflict | string | 内心冲突 |
| arc | string | 人物弧线描述 |
| catchphrase | string | 口头禅/标志性台词风格 |

### 3.3 Episode（单集）

| 属性 | 类型 | 说明 |
|------|------|------|
| episode_number | int | 集数 |
| title | string | 标题 |
| core_conflict | string | 本集核心冲突 |
| hook | string | 结尾钩子描述 |
| is_paywall | bool | 是否付费卡点集 |
| cliffhanger_type | enum | 悬念型/反转型/冲突升级型/情绪临界型 |
| scene_ids | list | 包含的场次编号 |
| estimated_duration_sec | int | 预估总时长 |

### 3.4 Scene（场次）

| 属性 | 类型 | 说明 |
|------|------|------|
| scene_id | string | 场次编号，如 "1-1"（集数-场次序号） |
| time | enum | 日 / 夜 / 晨 / 昏 |
| location_type | enum | 内 / 外 |
| location | string | 具体地点 |
| characters | list | 本场出场人物 |
| plots | list[Plot] | 剧情内容序列 |

### 3.5 Plot（剧情点）

| 属性 | 类型 | 说明 |
|------|------|------|
| sequence | int | 在本场中的顺序 |
| type | enum | action / dialogue / monologue / os |
| character | string \| null | 人物名（动作时为 null） |
| content | string | 具体内容 |
| emotion | string | 情绪标签 |
| duration_sec | int | 预估时长（秒） |
| is_cliffhanger | bool | 是否为卡点/钩子 |
| intensity | int | 情绪强度（1-5） |

### 3.6 KnowledgeDoc（知识库文档）

| 属性 | 类型 | 说明 |
|------|------|------|
| doc_id | string | 文档唯一标识 |
| title | string | 文档标题 |
| source_url | string | 来源链接 |
| category | enum | market_research / writing_method / platform_rules / case_study |
| published_date | date | 原文发布日期 |
| ingested_date | date | 入库日期 |
| rag_score | float | RAG 适用性评分 |
| content_md | string | Markdown 格式的清洗后内容 |
| tags | list | 分类标签 |

## 4. 工作流状态机

```
[草稿] → PARSE → [JSON数据] → RENDER → [格式化文本] → VALIDATE → [成品剧本]
                                                      ↓ (不通过)
                                                  [校验报告] → 人工修正 → [草稿]
                                                      ↓ (通过)
                                                  [存档] → [更新知识库]
```

---

## 5. 技术选型映射

| 功能模块 | 技术方案 | 说明 |
|----------|----------|------|
| 创意生成 + 草稿解析 | LLM API (GLM-5 / MiniMax) | Prompt 驱动，输出 JSON |
| RAG 知识库 | Google Vertex AI Search | 云端部署，语义检索 |
| 实时搜索 | Tavily API | 搜索市场数据、写作方法 |
| 格式渲染 | Python 脚本 | JSON → 工业标准格式 |
| 格式校验 | Python 脚本 | 正则 + 规则引擎 |
| 知识库文档格式 | Markdown + 元数据 JSON | 便于人读 + 机器解析 |
| 本地工作区 | 文件系统（drafts/scripts/knowledge_base） | 简单直接 |
