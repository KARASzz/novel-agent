# 红果短剧神经科学工作流 3.0 - 终极全链路蓝图

> 本文档深度融合了“红果商业投稿流（旧版业务主线）”与“大模型数据视图分离架构（2.0代码管线）”，提供从灵感、撰写、代码渲染到投递的端到端科学工作流水线，并详细规划了底层技术选型及落地实施路线。

---

## 1. 核心工业理念 (Core Philosophy)
在 3.0 体系中，**人类编剧的重心彻底回归“灵感创作与对焦平台商业要素”**，完全摒弃排版执念；**系统引擎负责“极客逻辑的质检与自动化排版”**。
系统秉承“数据与视图完全分离”核心思想：
- **人机共创（创意端）**：依托数据热力榜，提炼钩子与人物爽点。
- **大模型（理解抽象端）**：智能解析编剧写意粗放的草稿，提取情节、填补缺漏元数据，严格打包成底层 `SceneData` 的 JSON 零件。
- **Python（工业排版及质检端）**：按照平台（如《玫瑰冠冕》的标准）要求的极致排版准则重新拼接 JSON，并严格质检字数、反转悬念、时长红线。

## 2. 核心业务流水线 (Business Pipeline)

### 阶段一：灵感、定位与架构期 (Ideation & Architecture) —— 【人机共创】
**目标：打赢“前30秒”的立项战。**
- **市场调研与对标 (RAG 辅助)**：分析红果当前“热力榜”，接入 Tavily 搜索最新爆款动态。**优先寻找番茄小说完结 IP 评估短剧化可能**。
- **核心设定提炼 (Prompt 辅助引导)**：打磨一句话简介 (Logline)，明确核心卖点与受众的情绪价值。
- **故事架构**：设立主角极致反差感与反派打脸逻辑；千字内明确前30秒钩子与每集反转点设计。

### 阶段二：剧本草稿撰写期 (Drafting) —— 【人类创作】
**目标：超低心智负担输入，只管极致爽点输出。**
- **简易标记**：只需使用 `### 场景`、`【关键动作】`、`人名：` 等基础标识。
- **发力重点**：彻底去心理活动化，转化为具体的画面与表情机位；严格把控单集1-5分钟体量；保证台词精炼、句句带梗。

### 阶段三：自动化深度解析与渲染 (Automation Engine) —— 【AI与代码主导】
**目标：彻底根治大模型排版紊乱，实现短剧文本向工业流水件的无损转化。**
- **LLM 智能解析器 (Parser)**：大模型不直接扩写剧本，负责读懂凌乱的草稿，补齐缺失元数据（场景号推算、人物情绪推断、动作时长预估），输出结构化 JSON。
- **Python 格式渲染器 (FormatRenderer)**：提取 JSON，按平台规范渲染。如：强制居中对齐、动作说明前必须加 `▲`，对白自动下排。
- **Python 格式校验器 (FormatValidator)**：强逻辑排雷工具。单集时长超出2分钟抛出预警，扫描如果没有结尾动作/台词悬念钩子，则截停并打回重修。

### 阶段四：工业标准投稿包组装 (Submission Packaging) —— 【代码一键打包】
由自动批处理程序将几十集内容跑完后，自动组装为红果指定的投稿压缩包套装：
1. **[核心]** 核心概念与定位说明 (Pitch)
2. **[核心]** 完整人物小传
3. **[核心]** 完整故事大纲
4. **[核心]** 完美排版无短板的剧本正文（全剧或前30集囤稿）

### 阶段五：渠道投递与数据回馈 (Delivery & Iteration) —— 【迭代闭环】
- **应用发力点**：通过“番茄作家助手”App 投递，在问卷阶段着重展示历史成绩与有效产能。
- **知识回写库 (Knowledge Update)**：系统将承制方反馈和打回意见清洗入 RAG 库，优化下一个项目的避坑预判能力。

## 3. 技术落地路线图 (Technical Roadmap)

为落地上述宏大的业务流，系统代码层面的开发按如下阶段进行：

### 阶段 1：核心引擎构建 (MVP 阶段) [预计 1-2 周]
**任务：跑通“草稿 → JSON → 工业格式剧本”的最短核心链路。**
1. **Prompt 固化**：调试解析草稿的大模型 Prompt（选用 GLM-5 等长文本代码能力强的模型），使其能稳定吐出 `SceneData` JSON。
2. **构建 `renderer.py`**：纯 Python 实现 JSON 逆向拼装，精准排版并进行基础的“语速->绝对时长”算计。

### 阶段 2：商业质检与工业批量增强 [预计 2-3 周]
**任务：确保 100% 格式合规，植入审查级规则引擎。**
1. **构建 `validator.py`**：用正则验证全部格式的排版合法性，“前三集定律”与“付费卡点（如第15集）”规则植入断言扫描。
2. **构建 `batch_processor.py` 与 `main_pipeline.py`**：支持自动遍历工作区子目录，实现几十集多线程处理，产出合并报告。

### 阶段 3：RAG 智能前置驱动 [预计 2 周]
**任务：把短剧市场实时大盘与红果规则挂在 AI 模型脑子里。**
1. **知识基建**：接入 Tavily API 随时补充外网爆款库。
2. **全自动检索**：部署本地向量库（或 Google Vertex AI），在第一阶段生成创意时，让系统主动带上同题材爆款爽点作为参考输入给人类。

### 阶段 4：LTM 长期记忆网络（跨剧本经验库）[未来规划 4.0]
**任务：打造专属个人的短剧经验进化中枢。**
1. **跨剧本经验反刍**：基于阿里云百炼 LTM 2.0，提取每部完结剧本的成功卡点、写作习惯、避坑教训并作为长期记忆存储。
2. **写作管家 (Profile)**：随着单人使用时长的增加，系统深度绑定您的个性偏好，未来创作第 N 本剧本时，直接继承前 N-1 本的写作功力。

## 4. 技术栈与目录结构规范 (Architecture & Workspace)

**【技术栈选型】**
- **中枢语言**：Python 3.10+（环境极轻量化，高度依赖内置的 `re`, `json`，舍弃沉重框架）
- **智力引擎**：GLM-5 API (最懂中文互联网梗和代码格式)
- **知识挂载**：Google Vertex AI Search 或 轻量本地词法向量库混合检索引擎

**【物理目录严格约束】**
```text
ShortDramaWorkflow/
├── drafts/               # [输入] 专门吃人类编写的半成品草稿 (*.md/*.txt)
├── knowledge_base/       # [RAG支持] 短剧爆款写作技巧、平台封禁词与过稿规则 (*.md)
├── templates/            # [模板] 给人类参考的骨架草稿格式以及各种输入卡片
├── core_engine/          # [核心逻辑引擎]
│   ├── parser_prompts/   # 用于提取 JSON 的各类严苛 Prompt 模板
│   ├── schemas.py        # 约定俗成的数据结构定义验证文件
│   ├── renderer.py       # 格式终极渲染器 (生成带有 ▲ 排版的最终排布)
│   ├── validator.py      # 短剧质检官 (字数、时长、商业钩子合规性检查)
│   └── main_pipeline.py  # 主路由，一键串联所有批处理
├── reports/              # [反馈] 每次跑代码生成的预警、单集时长统计、评分表
├── scripts_output/       # [成品] 可以直接装进 ZIP 投递红果的顶级成品剧本库 (*.txt/*.docx)
└── logs/                 # [日志] 系统运行状态与报错日志追踪
```

## 5. 里程碑：第一步动作建议 (Next Action Item)

不需要着急写复杂的 LLM 提取逻辑！为了立证这套 **“分离渲染管线”** 的可行性，我们只需从 **“技术路线 阶段 1”** 的 `renderer.py` 原型开发开始：
**我们手工捏造一个符合约定的 JSON 测试案例丢进去，看是否能稳定输出含自动时长侦测、且带标准对位动作指令标号的完美版短剧文本！** 只要文本质感出来，这套理念便百分百跑通了。
